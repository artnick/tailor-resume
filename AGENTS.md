## Cursor Cloud specific instructions

`tailor-resume` is a pnpm + Turborepo monorepo (a "Resume Builder" app). Pinned to `pnpm@9.0.0` via corepack. Relevant workspaces:

| Workspace | Role | Dev command | Port |

| `apps/web` | Next.js 16 frontend (calls the API `/health`) | `pnpm --filter web dev` | 3000 |
| `packages/shared` | Zod schemas/types consumed by `api` | (built; see below) | â€” |

### Database (PostgreSQL) â€” required for the API

The API validates `DATABASE_URL` at startup and `/health` runs `SELECT 1`, so Postgres must be up. The repo's intended method is `docker compose up -d db`, but Docker is not preinstalled in this VM. The simplest working setup is native Postgres:

- Postgres is installed via apt but is **not auto-started** in this container. Start it with: `sudo pg_ctlcluster 16 main start` (verify with `sudo pg_lsclusters`).
- One-time DB bootstrap (idempotent-ish): set the password and create the DB:
  - `sudo -u postgres psql -c "ALTER USER postgres WITH PASSWORD 'postgres';"`
  - `sudo -u postgres psql -c "CREATE DATABASE resume_builder;"` (skip if it already exists)
- Apply schema + demo data: `pnpm --filter api db:migrate:deploy` then `pnpm --filter api db:seed`.
- `apps/api/.env` is gitignored. The update script copies it from `apps/api/.env.example` (DB on `localhost:5432`, user/pass `postgres`/`postgres`, db `resume_builder`).

### Non-obvious gotchas

- **`@repo/shared` must be built before `api` type-checks or runs standalone.** Its `package.json` `exports` point at `dist/` (gitignored). Either run `pnpm --filter @repo/shared build` once, or run `pnpm dev` at the root (Turbo runs the shared `tsc --watch` which emits `dist/`). Without it, `api` fails with `Cannot find module '@repo/shared'`.
- **Prisma client is not auto-generated by `pnpm install`.** Run `pnpm --filter api prisma:generate` (handled by the update script). The API will not boot without it.
- `turbo.json` sets `"ui": "tui"`, so root `pnpm dev` opens an interactive terminal UI. For background/log capture, run apps individually with `--filter` instead.

### Lint / type-check / test

- Lint: `pnpm lint`. Type-check: `pnpm check-types` (requires `@repo/shared` built â€” see above).
- Tests: there is no root `test` script; use `pnpm exec turbo run test`. `packages/shared` uses vitest (passes). `apps/api` uses jest but currently has no `*.spec.ts` files, so its `test` exits non-zero with "No tests found" â€” this is expected, not a setup failure.
- The Husky pre-commit hook runs `pnpm lint && pnpm check-types`.
